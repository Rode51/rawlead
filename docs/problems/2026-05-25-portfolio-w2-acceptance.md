### U5 — что за цифра 3?

**Сейчас в коде:** число **применённых** навыков (`appliedTags.length`) — сколько тегов уже влияют на ленту после «Применить».

**Проблема UX:** неочевидно; выглядит как «3 новых» или «3 в черновике».

**Решение Lead (согласовать с Design):**

| Вариант | Поведение |
|---------|-----------|
| **A (рекомендуем)** | Бейдж только если `appliedTags.length > 0`, подпись tooltip `title="Применено навыков: 3"` |
| B | Показывать **черновик** `draftTags.length` только пока `draft ≠ applied` (оранжевый) |
| C | Убрать бейдж, оставить только hint «Применить» |

**Владелец 2026-05-25:** вариант **A** — оставляем бейдж + `title="Применено навыков: N"`.

## Фильтры по вакансиям — волны (не всё сразу)

### Волна A — код, 0 ₽ (§ P7, Coder)

Жёсткая **категория** при парсинге из раздела биржи (FL/Kwork/…): «Логотипы» → `design`, не гнать категоризацию в LLM.

→ [`INGEST_CATEGORY_STRATEGY.md`](../team/architect/INGEST_CATEGORY_STRATEGY.md)

### Волна B — дешёвая LLM только для краткого текста (§ P8)

`deepseek-chat` / `llama-3-8b` для заголовка/сути/`ai_reasons` **коротко**; полный дорогой разбор — не на каждый лид.

**Lead:** расчёт ~130 ₽/день на 10k лидов — **реалистичен только** если в промпт идёт **snippet**, не простыня; вердикт «Брать» лучше оставить правила + `ai_score` или один дешёвый вызов с жёстким JSON.

### Волна C — UI фильтров по категориям (Designer)

Чипы «Дизайн · Разработка · Тексты…» в sidebar — **после** поле `category` в Neon и API `?category=`.

Спека: дополнить `feed-cabinet-mvp.md` §2.2 — Lead Design.

### Dogfood не ломать

| Контур | Лента / фильтры |
|--------|------------------|
| **Публичная** `/lenta/` | whitelist источников + категории для всех |
| **Владелец** | бот + опционально `/cabinet` или флаг `owner=1` — **все** источники, старые TG, без категорийного отсечения |

Реализация: отдельный query или `GET /v1/me/feed` для dogfood; бот без изменений логики notify.
