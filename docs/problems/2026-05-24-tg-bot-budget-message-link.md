## Ожидание (Coder § E)

| # | Готово когда |
|---|----------------|
| E1 | TG-пост **без бюджета в тексте** → после фильтра/ИИ **пересылается** как сейчас; в карточке бота: **`💰 по договорённости`** (не «не указан») |
| E2 | Если в тексте поста есть сумма — по возможности положить в `budget_text` (reuse `budget.parse_budget_rub` / простой regex); иначе «по договорённости» |
| E3 | В тексте карточки бота (TG-источник): **`📨 Сообщение #<id>`** (id из Telegram) |
| E4 | В карточке **кликабельная ссылка** на пост: `project.url` из `_message_url` — строка вида `🔗 Открыть сообщение` |
| E5 | Inline-кнопка бота ведёт на **это сообщение** (`project.url`), fallback invite — только если url пуст |
| E6 | FL/Kwork карточки **не сломаны** — «не указан» только где уместно для бирж |

---

## Приёмка

1. Группа prompt-test (или любой listen-чат): пост **без суммы**, с ключом «берём» (напр. «нужен парсер Python») **с чужого аккаунта**.
2. В боте: переслан оригинал + карточка с **по договорённости**, **#N**, ссылка открывает **этот** пост.
3. Пост **с суммой** в тексте → в карточке сумма, не «по договорённости».
4. FL/Kwork — smoke одной карточки без регрессии.

## Решение (2026-05-24)

| Файл | Изменение |
|------|-----------|
| `src/budget.py` | `extract_budget_text_from_post()` — regex + `parse_budget_rub` |
| `src/tg_monitor.py` | `budget_text` из текста поста |
| `src/telegram_notify.py` | TG: «по договорённости»; `📨 Сообщение #id`; ссылка/кнопка → `project.url` |

**Как проверить:** пост без суммы в listen-чате → пересыл + карточка с «по договорённости», `#N`, кликабельный `t.me/c/.../N`. Пост с «15 000 руб» → сумма в карточке.

**Пересылка (2026-05-24):** acc → **бот** (не в личку владельца от acc); бот `forwardMessage` в `TELEGRAM_CHAT_ID`. На acc — `/start` у бота.
