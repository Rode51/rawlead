**Статус:** ✅ принято владельцем 2026-05-27  
**Дата:** 2026-05-27  
**Приоритет:** P0 (dogfood / PRE-LAUNCH C4)

## Симптом (факт владельца)

- В @FLPARSINGBOT приходит полный **Разбор** (вердикт **Сомнительно**), блок **«Черновик отклика»** пустой (только разделители и ссылка).
- Пример: Kwork `3184374`, ~22:51 — в `data/radar_legacy.log`: `neon:цикл │ … │ в бот 1`.

## Причина (Lead, код + лог)

1. **Промпт legacy** (`src/ai_analyze.py` system, канон `AI.md` § legacy): `reply_draft` **только при «Брать»**; при **«Сомнительно»** модель отдаёт `""` (парсер сбрасывает `-` / `—`).
2. **Формат TG в скрине** — заголовок «Черновик отклика **(подправь под себя)**» — в **текущем** `telegram_notify.py` такой строки **нет** (только «Черновик отклика:»). Значит воркер, скорее всего, крутится на **старом коде в памяти** (не перезапускали Legacy ▶ после правок Coder).
3. **В repo уже есть C4:** `process_legacy_neon_listing` + `send_listing_notification` — без непустого `reply_draft` **не** слать; в логе должно быть `neon:skip … skip:ai:no_reply_draft`. После перезапуска consumer пустые «Сомнительно» **не должны** приходить.
