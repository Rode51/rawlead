# Health-check мониторингового аккаунта

## Зачем

Раз в **15 мин** (настраивается) радар проверяет, что **+66** (Telethon) на связи. При сбое — сообщение **в твоего бота**:

```
⚠️ Аккаунт +66953964608 не отвечает

Причина: …
```

Когда снова ок — одно сообщение **✅ снова на связи**.

## Когда срабатывает

| Процесс | Как часто |
|---------|-----------|
| `src/main.py` (биржи) | раз в `HEALTH_CHECK_MINUTES` в главном цикле |
| `scripts/tg_main.py` (TG) | вместе с `тг:пульс` (~каждые 2 мин, но проверка Telethon — по интервалу) |

Повтор того же сбоя в бот — не чаще **`HEALTH_ALERT_COOLDOWN_MINUTES`** (по умолчанию 60).

## Ручная проверка

```powershell
cd C:\Users\hramo\uisness
.venv\Scripts\python.exe scripts\tg_health.py
```

`OK` / `FAIL` в консоли; при FAIL — уведомление в бот (если бот сам доступен).

## `.env`

```env
HEALTH_CHECK_MINUTES=15
HEALTH_ALERT_COOLDOWN_MINUTES=60
```

## Лог

`data/radar.log`:

- `здравье:ок` — аккаунт отвечает
- `здравье:сбой` — не отвечает
- `здравье:уведомление отправлено` — писали в бот
