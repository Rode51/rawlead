# Мониторинговые аккаунты Session+Json

**Важно:** в эти номера **нельзя** войти через смартфон (код по SMS не придёт / аккаунт только под софт).

Формат покупки: **SESSION + JSON** — работа только через программы под такие аккаунты или через **Telethon** в нашем радаре (фаза 1, подключит Coder).

Официальная памятка продавца:  
https://retriv.market/help/telegram/kak-avtorizovatsmya-v-telegram-versii-session-json-sessionjson

---

## Что лежит у тебя на ПК

Обычно в одной папке (у тебя `Desktop\Parser`):

| Файл | Назначение |
|------|------------|
| `+669….session` | Сессия Telethon — **её** укажем в `.env` → `TELETHON_SESSION_PATH` |
| `+669….json` | Пара к сессии (данные аккаунта для софта продавца) — **храни рядом**, в Git не класть |

Три номера = три пары. Сейчас все под **мониторинг**; в коде активен один путь в `.env` (остальные — позже).

## Как «зайти» и вступить в чаты (без телефона)

### Вариант A — бесплатно: Telethon (рекомендуем)

| Способ | Что это | Вступить в чаты | Слушать заказы |
|--------|---------|-----------------|----------------|
| **Telethon** (Python) | Библиотека MIT, $0 | Да — по ссылке-приглашению скриптом | Да — это наш радар (фаза 1) |
| **Pyrogram** | Аналог Telethon, $0 | То же | То же |
| Платный GUI | Prime и т.д. | Да, окном программы | Иногда |

**Практично для тебя:** не покупать софт — использовать **уже купленные** `.session` + бесплатный **Telethon**:

1. Один раз в `.env`: `TELEGRAM_API_ID`, `TELEGRAM_API_HASH` (my.telegram.org).
2. Coder даст короткий скрипт **`join`** (вступить по ссылкам) и потом **мониторинг** в радаре.
3. Файл `.json` для Telethon **не обязателен** — достаточно `.session` + прокси.

Пока скрипта join нет — либо ждать Coder после «готов пул TG», либо временно платный GUI (если очень срочно вступить в чаты).

**Отдельного бесплатного аналога Prime с окном** под Session+Json почти нет — либо платные/триал, либо скрипты.

### Вариант B — платный GUI (если срочно вступить в чаты до скрипта)

Prime, TG-GIANT, Telegram Gods — обычно **платно или триал**. Retriv — ссылка вверху.

Типичный порядок:

1. Установить софт с сайта разработчика (не из случайных ссылок).
2. **Импорт** аккаунта: указать `.session` + `.json` (или папку аккаунта).
3. В настройках аккаунта в софте — **прокси** (тот же, что в `.env` → `TELETHON_PROXY_URL`).
4. Через этот софт **вступить в 3–5 чатов** по ссылке (как в обычном TG, но только в окне софта).
5. Записать чаты в [`SOURCES_POOLS.md`](SOURCES_POOLS.md).

**Не** пытайся «Добавить аккаунт» в обычном Telegram на телефоне — для Session+Json это не работает.

### Вариант C — наш радар (скоро)

После `docs/team/TZ_TG.md` Coder подключит **Telethon**: читает твой `.session` + прокси, слушает чаты, шлёт лиды в **бота** (личный чат).

Вступление в **новые** чаты тогда:

- либо снова через софт (вариант A),
- либо отдельной командой/скриптом (если добавим в ТЗ — «join по ссылке»).

## Настройка `.env` для радара (не смартфон)

1. [my.telegram.org/apps](https://my.telegram.org/apps) — один раз получить (логин **своим** личным TG, не мониторинговым):

   ```
   TELEGRAM_API_ID=...
   TELEGRAM_API_HASH=...
   ```

2. Прокси мониторинга (уже есть):

   ```
   TELETHON_PROXY_URL=http://host:port:user:pass
   ```

3. Путь к сессии **без** расширения `.session`:

   ```
   TELETHON_SESSION_PATH=C:/Users/hramo/Desktop/Parser/+66953964608_telethon
   ```

4. Если при `tg_list_dialogs.py` ошибка `no such column: version` — файл **Pyrogram**, не Telethon. Конвертация:

   ```powershell
   .venv\Scripts\python.exe scripts\tg_convert_session.py C:\Users\hramo\Desktop\Parser\+66953964608
   ```

   В `.env` укажи путь к `*_telethon` (без `.session`).

5. Файл `.json` с тем же именем держи **в той же папке** — для софта; радар Telethon обычно нужен только `.session`.

Переключить активный номер — поменяй `TELETHON_SESSION_PATH` на другой `+669…`.

## Два канала — не путать

| Что | Как |
|-----|-----|
| **Уведомления тебе** (лиды) | Личный Telegram + **бот** (`TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`) — [`RUN.md`](RUN.md) |
| **Мониторинг чатов** | Session+Json + прокси + софт **или** будущий Telethon в `src/` |

Личный аккаунт и три купленных — **разные миры**.

---

## Частые ошибки

| Ошибка | Почему |
|--------|--------|
| Ввожу номер в Telegram на телефоне | Session+Json так не авторизуется |
| Удалил `.session` | Аккаунт «умер» для софта и радара |
| Нет прокси в софте | Риск бана / не коннектится |
| Нет API_ID/HASH в `.env` | Telethon не запустится, даже с живой сессией |
