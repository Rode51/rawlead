# Настройка выбора проектов

## Как пользоваться

Радар **читает этот файл из кода**. Правишь списки → сохраняешь → перезапуск `python src/main.py`.

Широкая лента: в `.env` **`FILTER_WIDE=1`** (почти всё в ИИ, режет только **стоп** ниже). Узко: `FILTER_WIDE=0` — нужно слово из **«Берём»**.

URL бирж — [`SOURCES.md`](SOURCES.md). Профиль для ИИ — [`PROFILE.md`](PROFILE.md).

---

## Уровень 1 — какие ленты FL смотреть

В файле **`.env`**:

| Переменная | Что делает |
|------------|------------|
| `FL_PROJECTS_URL` | Одна страница категории (или «все проекты») |

Примеры URL — в `docs/ops/SOURCES.md`.

**Этап 1.5:** несколько URL через запятую (`FL_PROJECTS_URLS`) — Coder, когда скажешь.

## Уровень 2 — ключевые слова (быстрый отсев)

Программа смотрит **заголовок** и **краткое описание** с листинга.

**Режим в `.env` → `FILTER_WIDE`:**

| Значение | Логика |
|----------|--------|
| **`1` (рекомендуется)** | Пропускаем в ИИ **все новые** заказы, кроме явного **стопа** из списка ниже. ИИ сам решает «потяну ли с Cursor». |
| **`0` (узко)** | Нужно **хотя бы одно** слово из «берём» **и** нет стопа — много заказов до ИИ не доходит. |

По умолчанию в коде **`FILTER_WIDE=1`** — широкая лента, отсев делает нейросеть.

Списки ниже — **боевые** (подбор через Deep Research). Дальше по факту ленты: добавляй стопы или уточняй «берём», пока Telegram не станет «твоим».

### Берём (белый список)

**Парсинг и данные:** парсер, парсинга, спарсить, сбор данных, выгрузить товары, выгрузка, скачать базу, сбор контактов

**Боты, скрипты, таблицы:** бот, бота, ботов, телеграм бот, tg бот, скрипт, автоматизация, макрос, макросы, excel, таблицы

**Маркетплейсы и карточки:** карточки товаров, наполнение карточек, wb, wildberries, ozon, озон, заведение карточек

**Сайты:** лендинг, визитка, html, css, сайт-визитка

**Тексты и CMS:** исправить текст, копирайтинг, рерайт, статья, написать текст, seo текст, наполнение сайта, wordpress, вордпресс, tilda, тильда

### Отсекаем (чёрный список — если есть, в Telegram **не** шлём)

- логотип, баннер, дизайн макета, отрисовать, figma, фигма, вектор, 3d, модель, анимация, фотошоп, photoshop, видеомонтаж, смонтировать, c++, c#, swift, ios, android, 1с, дипломная, курсовая, реферат, решить задачу, контрольная, накрутка, отзывы, написать отзыв, клики, лайки, регистрация аккаунтов, прогрев, капча

**Код:** `src/filters.py` → `docs/ops/FILTERS.md` (`default_listing_filter()`). Подстроки: стоп → отказ; при `FILTER_WIDE=1` без стопа → в ИИ.

**Ограничение:** совпадение по подстроке — стопы подбирай короткими, без ложных вроде `java` в `javascript` (убрали `java` из списка).

**Как менять:** правка списков в этом файле → сохранить → перезапуск `python src/main.py`.

## Уровень 4 — разбор ИИ (этап 2)

Только проекты, прошедшие уровни 1–3. ИИ читает **полное описание** со страницы проекта + `docs/ops/PROFILE.md`.

См. **`docs/team/TZ.md` §5**.

---

## Логика целиком

```
Новый проект на FL
  → уже видели? (база) → стоп
  → лента из твоего URL в .env
  → есть «берём» и нет «стоп»? → иначе стоп
  → бюджет ≥ MIN? → иначе стоп (этап 3)
  → ИИ: брать / нет + разбор → сообщение в Telegram
```
