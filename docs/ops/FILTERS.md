# Настройка выбора проектов

## Как пользоваться

Радар **читает этот файл из кода**. Правишь списки → сохраняешь → перезапуск `python src/main.py`.

Широкая лента: в `.env` **`FILTER_WIDE=1`** (почти всё в ИИ, режет только **стоп** ниже). Узко: `FILTER_WIDE=0` — нужно слово из **«Берём»**.

URL бирж — [`SOURCES_POOLS.md`](SOURCES_POOLS.md) · архив деталей [`../archive/SOURCES.md`](../archive/SOURCES.md). Профиль для ИИ — [`PROFILE.md`](PROFILE.md).

**Deep Research (стоп-списки, таксономия, тесты):** [`FILTERS_DEEP_RESEARCH_2026.md`](../team/archive/FILTERS_DEEP_RESEARCH_2026.md) — **владелец не пишет списки вручную**; правки после dogfood → Coder § F-PROMPT / F-RESEARCH.

## Уровень 2 — ключевые слова (быстрый отсев)

Программа смотрит **заголовок** и **краткое описание** с листинга.

**Режим в `.env` → `FILTER_WIDE`:**

| Значение | Логика |
|----------|--------|
| **`1` (рекомендуется)** | Пропускаем в ИИ **все новые** заказы, кроме явного **стопа** из списка ниже. ИИ сам решает «потяну ли с Cursor». |
| **`0` (узко)** | Нужно **хотя бы одно** слово из «берём» **и** нет стопа — много заказов до ИИ не доходит. |

По умолчанию в коде **`FILTER_WIDE=1`** — широкая лента, отсев делает нейросеть.

Списки ниже — **боевые** (подбор через Deep Research). Дальше по факту ленты: добавляй стопы или уточняй «берём», пока Telegram не станет «твоим».

## Логика целиком

```
Новый проект на FL
  → уже видели? (база) → стоп
  → лента из твоего URL в .env
  → есть «берём» и нет «стоп»? → иначе стоп
  → бюджет ≥ MIN? → иначе стоп (этап 3)
  → ИИ: брать / нет + разбор → сообщение в Telegram
```
