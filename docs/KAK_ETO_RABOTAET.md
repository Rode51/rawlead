## Зачем это

Программа **сама** смотрит заказы на **FL.ru**, **Kwork** и в **Telegram-чатах**, отбирает подходящие, спрашивает **ИИ** «стоит ли откликаться». **Из TG** — сначала **пересылает оригинал поста** в личку (Telethon), затем бот присылает **разбор** (как с бирж). Ссылки на пост в тексте не нужны — открываешь пересланное.

Ты **не сидишь** в десятках чатов — читаешь только отобранное.

### Три канала продукта (vision v0.11)

| Канал | Где | Для кого | Деньги |
|-------|-----|----------|--------|
| **1. Dogfood** | @FLPARSINGBOT (legacy) | Ты — ловишь заказы и откликаешься | Личный ROI |
| **2. Открытая лента** | Сайт `/lenta/` | Любой гость | Бесплатно — приманка |
| **3. ИИ-агент** | `/cabinet/` + подписка | Фрилансер с тегами | **590–990 ₽/мес** (после биллинга) |

Воронка: лента → регистрация (match по навыкам) → платная подписка (черновик отклика, push). Подробно: [`team/product/PRODUCT_VISION.md`](team/product/PRODUCT_VISION.md).

**Сейчас в коде:** лента и кабинет живые; L2 в кабинете есть; **оплата и push подписчику** — фаза 3f/3h (см. [`team/architect/ROADMAP.md`](team/architect/ROADMAP.md)).

## Как часто что проверяется (2026-05-28)

| Источник | Как | Интервал | Куда попадает |
|----------|-----|----------|----------------|
| **FL.ru + Kwork** | Site `main.py`, конвейер | **~1 мин** между циклами fetch (`POLL_INTERVAL_MINUTES=1`, `RADAR_CONVEYOR=1` в `.env.site`) | Neon → L1 батч → `is_visible` → `/lenta/` |
| **Telegram** | Site `tg_main`, **постоянно** (не по минутам) | Событие = новый пост в чате | Neon + L1 → `/lenta/` (если `source` в `PUBLIC_FEED_SOURCES`) |
| **Join новых чатов** | Внутри `tg_main` | ~4/час на acc, ночь 02:00–07:00 Irkutsk — пауза | `TG_JOIN_QUEUE_v2.csv` → listen |

**Важно:** «10 минут» в старых текстах — это был **пауза между целыми циклами** (fetch+L1 на каждой карточке). С **конвейером** fetch бирж — **каждую минуту**, L1 — **пачкой** в конце цикла (`L1_BATCH_PER_CYCLE`, по умолчанию 40).

**Проверка «живо ли»:** `data/radar_site.log` — заголовок `── Цикл … ──` каждые ~1–3 мин; `site:сводка │ 10мин` — здоровье за 10 мин; `тг:пульс` — TG жив (раз в ~2 мин).

## L1 и L2 — кто за что отвечает

```
Заказ (FL/Kwork/TG)
  → словесный фильтр (FILTERS_SITE / LEGACY)
  → Neon + SQLite
  → L1 (analyze_lite) — вердикт, score, lead_tags, task_summary, is_visible для ленты
  → /lenta/  — все с is_visible=true (бесплатная витрина)

Подписчик в /cabinet/:
  → match по user_tags (final_rank)
  → L2 (analyze_premium) — по кнопке «Написать отклик» / me/feed с reply_draft
  → черновик отклика + рыночная цена (персонализация — § P4b)
```

| Уровень | Где | Что даёт | Кому |
|---------|-----|----------|------|
| **L1** | Ingest Site | «Брать / Сомнительно / МИМО», теги, видимость в **открытой** ленте | Все гости `/lenta/` |
| **L2** | API `/v1/me/feed`, кабинет | `reply_draft`, tools, углублённый разбор | **Подписчик** (после auth) |

**Legacy-бот** использует **один** полный разбор (`analyze_project`), не split L1/L2 — это отдельный контур dogfood.

Канон ИИ: [`team/architect/AI.md`](team/architect/AI.md).

| Часть | Простыми словами |
|------|------------------|

## Telegram — важно не путать

| Что | Объяснение |
|-----|------------|
| **Бот** | Только **уведомления** тебе (пауза, кнопки). В чужие чаты бот **не** заходит |
| **acc1, acc2, acc3** | Одно окно **TG** — каждый слушает свой файл `telethon_chat_ids_accN.txt` |
| **Уведомления** | Только от **людей**; сообщения **ботов** чата (верификация, приветствия) — пропуск |
| **Формат** | Пересылка + разбор с подписью **`acc2 · название чата`** (и acc1, acc3) |
| **Прокси** | У каждого номера свой IP |

**Список «куда вступить»:** [`ops/TG_JOIN_QUEUE.csv`](ops/TG_JOIN_QUEUE.csv)  
**Что слушает каждый acc:** `data/telethon_chat_ids_acc1.txt` (и `_acc2`, `_acc3`; пополняются после join)

**Сейчас в очереди (2026-05-23):** 14 чатов уже **`done`** (MVP + волна 2) + **73** новых **`pending`** (волна 3 из Google Doc).

## Какой номер в какой чат (принцип)

| Вопрос | Ответ |
|--------|--------|
| Бот подписывается? | **Нет.** Вступают только **acc1 / acc2 / acc3** (купленные номера). |
| Как назначается номер? | В CSV колонка **`account`**. Join-скрипт берёт только **свои** строки (`pending` + этот acc). |
| Как попало в CSV при волне 3? | **По очереди «карусель»:** 1-я новая ссылка → acc1, 2-я → acc2, 3-я → acc3, 4-я → снова acc1… (скрипт `tg_queue_import.py`). **Не** «все IT на acc2» — тематика в Doc на распределение не влияет. |
| Сколько на номер сейчас | acc1: **25** pending, acc2: **24**, acc3: **24** (всего 73). |

Полный реестр ~150 каналов в [Google Doc](https://docs.google.com/document/d/1KplF0v2eBr_o0ThbON17sRnfDqoMfiIkk_M3HkLtjfE/edit?usp=sharing); в очередь попали **ссылки из Doc** (без §7 «сомнительные»). Строки «Проверить вручную» без ссылки — пока не в очереди.

## Как само вступает в чаты (join)

1. CSV: **`pending`** + колонка **`account`** (acc1 / acc2 / acc3).
2. **Всё в одном окне TG** (`TG_JOIN_IN_TG_MAIN=1`): каждый номер раз в ~час вступает **только в свои** pending (паузы 15–20 мин).
3. Лимиты **на каждый** номер: **4**/час, **25**/сутки; ночь **02:00–07:00** Иркутск — без join.
4. После вступления → **`done`**, `chat_id`, чат сразу в listen (без отдельного процесса join).
5. Лог: `data/tg_join.log`.

**Нельзя** второй раз открыть ту же сессию (`tg_join_queue --account acc1`), пока пульт/TG запущен — `database is locked`.
