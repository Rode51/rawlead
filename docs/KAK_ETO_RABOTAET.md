# Как работает FL Radar (простыми словами)

Для владельца **без программирования**. Обновляет **Lead** при новых фичах.

Короткие шаги «что делать» — в [`FOR_YOU.md`](FOR_YOU.md).

---

## Зачем это

Программа **сама** смотрит заказы на **FL.ru**, **Kwork** и в **Telegram-чатах**, отбирает подходящие, спрашивает **ИИ** «стоит ли откликаться» и **присылает тебе в Telegram-бота** карточку с ссылкой.

Ты **не сидишь** в десятках чатов — читаешь только отобранное.

---

## Запуск на компьютере

**Обычно:** двойной клик **`scripts\start-radar-desktop.bat`** → окно пульта → **▶ Старт**.  
Три программы работают **в фоне** (без чёрных cmd): биржи, TG, join.

| Часть | Простыми словами |
|------|------------------|
| **Биржи** | FL.ru + Kwork по расписанию |
| **TG** | Три номера читают свои чаты; acc1 ещё вступает в новые |
| **Join** | acc2/acc3 вступают в чаты из очереди |

**⏹ Стоп** в пульте — всё гасит. Запасной способ: `start-radar-full.bat` (3 cmd) — [`ops/DESKTOP_LAUNCH.md`](ops/DESKTOP_LAUNCH.md).

## Telegram — важно не путать

| Что | Объяснение |
|-----|------------|
| **Бот** | Только **уведомления** тебе (пауза, кнопки). В чужие чаты бот **не** заходит |
| **acc1, acc2, acc3** | Окно **TG** (`tg_main`) **слушает** свои чаты (файл `data/telethon_chat_ids_accN.txt` на аккаунт) |
| **acc1** | Плюс **сам вступает** в новые чаты из очереди (внутри того же окна) |
| **acc2, acc3** | **Вступают** в чаты в окне join; после `done` id попадает в свой listen-файл |
| **Прокси** | У каждого номера свой IP — чтобы Telegram не связал с личным аккаунтом |

**Список «куда вступить»:** [`ops/TG_JOIN_QUEUE.csv`](ops/TG_JOIN_QUEUE.csv)  
**Что слушает каждый acc:** `data/telethon_chat_ids_acc1.txt` (и `_acc2`, `_acc3`; пополняются после join)

## Как само вступает в чаты (join)

1. В таблице CSV строки **`pending`** — ещё не вступили.
2. **acc1:** окно **TG** раз в час проверяет очередь, вступает аккуратно (паузы 15–20 мин, ночью 02:00–07:00 Иркутск — пауза).
3. **acc2/acc3:** окно **join** (один supervisor) делает то же для своих строк.
4. После вступления в CSV ставится **`done`** и пишется `chat_id`.
5. Новый чат попадает в listen-файл своего acc; **acc1** подхватывает без рестарта TG; acc2/acc3 — перезапуск окна TG или `tg_sync_chat_ids.py`.

Лог: `data/tg_join.log` (раз в день заглянуть).

**Нельзя** вручную запускать старую команду join для acc1, пока работает окно TG — будет ошибка «database is locked».
