|--|-----------|------------|

### 2.2 Filter Bar — горизонтальная плашка

Filter bar — sticky под header, всегда видна. Высота 52px desktop / 48px mobile.

```
┌──────────────────────────────────────────────────────────┐
│  [Все] [Разработка] [Дизайн] [Маркетинг] [Тексты]        │
│                              │ [Навыки ▾] [Сортировка ▾] │
└──────────────────────────────────────────────────────────┘
```

**Левая группа — категории (специализация):**

| Chip | API slug | Иконка |
|------|----------|--------|
| Все | — (пусто) | — |
| Разработка | `dev` | `</>` |
| Дизайн | `design` | `✦` |
| Маркетинг | `marketing` | `◎` |
| Тексты | `text` | `Aa` |

- Активный chip: bg `#4F46E5`, text `#FFFFFF`
- Неактивный: bg `#F3F3EF`, border `#E4E4E7`, text `#3F3F46`
- Смена категории → сброс `offset=0`; навыки — НЕ сбрасываются (они независимы)

**Правая группа — dropdown-кнопки:**

`[Навыки ▾]` — открывает panel/sheet с навыками из каталога  
`[Сортировка ▾]` — открывает mini-dropdown: «Новые» / «По совместимости»

Стиль dropdown-кнопок: bg `#F3F3EF`, border `#E4E4E7`, text `#3F3F46`, radius 999px; активный (есть выбор): bg `#EEF2FF`, border `#4F46E5`, text `#4F46E5`

### 2.3 Навыки — dropdown-panel (из каталога)

Открывается по клику `[Навыки ▾]`.

**Desktop:** выпадающая панель ниже кнопки, `max-width: 420px`, shadow, radius 16px, фон `#FFFFFF`  
**Mobile:** bottom sheet (translateY 300ms ease-out), overlay `rgba(0,0,0,0.4)`

**Внутри панели:**

```
┌─ Навыки ────────────────────────── [✕] ─┐
│                                          │
│  По специализации Дизайн:                │
│                                          │
│  [UI/UX дизайн] [Веб-дизайн] [Figma]    │
│  [Лендинг] [Мобайл] [Баннеры]           │
│  [Фирменный стиль] [Презентации]         │
│  [Логотипы]                              │
│                                          │
│  ╰ [Ещё навыки ▾]                          │
│                                          │
│              [Применить]  Сбросить       │
└──────────────────────────────────────────┘
```

**Правила контента:**

- Если выбрана категория → показываем `shows_in_ui=yes` навыки **только этой** ниши (Tier A)
- «Все» категории → показываем самые популярные Tier A (по 4–5 от каждой ниши)
- Кросс-нишевые навыки (`telegram_bot_dev` etc.) показываем в каждой нише где они указаны в каталоге
- **«Ещё навыки ▾»** → expand с Tier B (`tier_b_only`); в развёрнутом виде — **«Свернуть»**

**UX анона (`/lenta/`):**
- Выбор навыков → **только сортировка** (заказы с совпавшими тегами поднимаются вверх)
- Лента **никогда не пустеет** — все лиды остаются, только порядок меняется
- Подпись кнопки «Применить» — «Применить (сортировка по совместимости)» → или просто «Применить» с tooltip

**Chip навыка:**
- Активный: bg `#EEF2FF`, text `#4F46E5`, border `#C7D2FE`
- Неактивный: bg `#F3F3EF`, border `#E4E4E7`, text `#3F3F46`

**Глоссарий UX (использовать в тексте панели и tooltip):**

| Термин | Смысл для пользователя |
|--------|------------------------|
| Специализация | В какой нише ищешь заказы |
| Навыки | Что умеешь — влияет на сортировку |
| Совместимость | Насколько заказ похож на твой профиль |
| Применить | Уточнить порядок ленты по навыкам |
| Ещё навыки | Дополнительный список (Tier B) — узкие специализации, не показываем сразу чтобы не перегружать |

### 2.5 Заголовок ленты

```

```

- H1: Manrope 28px / 800, `#18181B`

### 2.6 Infinite scroll

## 5. Report bug — FAB

**Назначение:** пользователь сообщает об ошибке (неправильная карточка / не открывается / скрин).

```
Размещение: fixed bottom-right, bottom: 24px, right: 20px
Вид: круглая кнопка 40×40px, «?», bg #FFFFFF, border 1px #E4E4E7, shadow, #4F46E5 text
Mobile: bottom: 80px (над thumb-zone), right: 16px
```

**По клику — modal/sheet:**

```
┌─ Сообщить об ошибке ─────────────── [✕] ─┐
│                                            │
│ Что случилось?                             │
│ ┌──────────────────────────────────────┐  │
│ │ Опиши кратко...                      │  │
│ └──────────────────────────────────────┘  │
│                                            │
│ Страница: /lenta/ (подставляется авто)     │
│                                            │
│ [Отправить]                                │
└────────────────────────────────────────────┘
```

| Поле | Обязательно | Примечание |
|------|-------------|------------|
| Текст ошибки | да | textarea, min 10 символов |
| URL (авто) | нет | берётся из `window.location.href` |

**После отправки:** «Спасибо! Посмотрим.» + автозакрытие через 2с  
**Куда отправлять:** `POST /v1/feedback` (задача Coder; форма Coder-ом, не Designer)

**Scope:** WP-сайт на телефоне. Канон: LEAD_DESIGN_PROMPT § C1-MOBILE-UX.

### 7.1 Touch targets

- Все интерактивные элементы: `min-height: 44px`, `min-width: 44px`
- Кнопки в карточке: min-height `44px`
- Chip навыка: min-height `36px` (допустимо в плотной группе)

### 7.2 Bottom sheets — спека

| Sheet | Тригер | Размер | Содержимое |
|-------|--------|--------|------------|
| **Навыки** | `[Навыки▾]` в filter bar | Full-sheet `95vh` | Табы категорий → чипы по нише → sticky [Применить →] 52px |
| **Сортировка** | `[Сорт▾]` в filter bar | Half-sheet `40vh` | 2 radio варианта: «Новые» / «По совместимости» |
| **Навыки кабинета** | tap на collapsed skills row | Full-sheet `95vh` | autocomplete input + текущие чипы + [Сохранить] |

**CSS (все sheets):**
- `position: fixed; bottom: 0; left: 0; right: 0`
- `border-radius: 4px 4px 0 0` (NEO: без скругления на bottom)
- Handle: `width 40px; height 4px; bg var(--rl-border-light)` — по центру сверху
- Overlay: `rgba(0,0,0,0.5)` → tap закрывает
- Transform: `translateY(100%)` → `translateY(0)`, 280ms ease-out
- `overscroll-behavior: contain`

### 7.3 TWO-SPEEDS strip (mobile)

```
┌─────────────────────────────────────────────────────┐
│ ⏱ Обновляется раз в 15 мин · Подробнее →            │
└─────────────────────────────────────────────────────┘
```

- Позиция: под `.rl-filter-bar`, **не sticky**
- Стиль: Manrope 13px, `var(--rl-text-muted)`, padding `8px 16px`
- «Подробнее →» → ghost link `/pricing/`
- Desktop: та же строка (не sticky на обоих)

### 7.4 Hero mobile (лендинг)

```
┌─────────────────────────────────────────────────────┐
│                                                      │
│  Лиды без шума                  ← H1 32px/800        │
│                                                      │
│  Биржи и Telegram —             ← 16px/400           │
│  в одной ленте. ИИ убирает      ← 2 строки          │
│  мусор до тебя.                                      │
│                                 ← gap 32px           │
│  [Смотреть ленту →]             ← full-width 52px    │
│                                 ← gap 12px           │
│  [Смотреть тарифы ↓]            ← full-width 48px    │
│                                                      │
└─────────────────────────────────────────────────────┘
```

- `padding-top: 24px` (не 80px как desktop)
- CTA stack: `display: flex; flex-direction: column; gap: 12px`
- Live preview: **2 карточки** под hero, не pointer-events

### 7.5 TG Login кнопка (все состояния)

```html
<a href="tg://resolve?domain=rawlead_bot" class="rl-tg-login-btn">
  Войти через Telegram →
</a>
```

- Class: `.rl-tg-login-btn` — full-width, `min-height: 52px`, NEO primary style
- **Без** `<script async src="https://telegram.org/js/telegram-widget.js">` — не используем
- Fallback для desktop: та же кнопка, `max-width: 400px`
