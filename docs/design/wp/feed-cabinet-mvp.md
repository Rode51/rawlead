## 0. Принцип двух поверхностей

| | `/feed` | `/cabinet` |
|--|---------|------------|
| **Доступ** | anon, без регистрации | только авторизованный (`user_id=1` MVP) |
| **Лента** | все `is_visible=true` лиды | только match по `user_tags`, sorted by `final_rank` |
| **Карточка** | закрытая → раскрытие: текст + теги + ссылка | то же + кнопка «Сгенерировать отклик» + рыночная цена |
| **Ощущение** | «открытый рынок заказов» | «мой персональный поток» |

Обе страницы — **продуктовые**, не маркетинговые. Тот же язык REFERENCE E, но ощущение отдельного приложения (Linear / Superhuman tier).

### 2.2 Sidebar (sticky, desktop)

- Ширина: **280px**, sticky при скролле ленты
- Заголовок «Фильтры» — Manrope 14px/600
- Разделители секций — тонкие линии `#E8E8EC`
- Состояние filter chip при выборе: фон `#0A0A0A`, текст белый (как primary btn)
- «Сбросить фильтры» — ссылка-ghost под блоком, появляется только если выбран хоть один фильтр

**Фильтры:**
| Группа | Тип | Значения |
|--------|-----|---------|
| Источник | radio + «Все» | Все · FL.ru · Kwork · Telegram |
| Бюджет от | range slider | 0 — 100 000 ₽ |
| ИИ-оценка | radio | Любая · Брать (≥70) · Уверенно (≥85) |
| Сортировка | radio | Новые первыми · По оценке |

### 2.3 Mobile (< 768px)

- Sidebar скрыт. Над лентой — кнопка **«Фильтры ↑»** (pill, outline)
- По клику: **bottom sheet** выезжает снизу (transform translateY, 300ms ease-out)
- Bottom sheet: те же фильтры, кнопка «Применить» (primary) + «Сбросить» (ghost)
- Overlay: `rgba(0,0,0,0.4)` на фоне, клик по оверлею закрывает sheet

### 2.4 Заголовок ленты

```
Лента заказов                              847 лидов за 7 дней
```
- H1 Unbounded 32px / 900
- Счётчик — Manrope 14px muted, справа (desktop) или под заголовком (mobile)

### 2.5 Infinite scroll

- При достижении sentinel (последний элемент в 80% viewport) — подгрузка следующей страницы
- Во время загрузки: **3 skeleton-карточки** (серые блоки, pulse-анимация)
- Если данных больше нет: текст «Все заказы загружены» центрированно, muted цвет

## 3. Карточка лида — главный компонент

### 3.1 Закрытое состояние

```
┌──────────────────────────────────────────────────────┐
│ [●FL.ru]  Верстка лендинга на WordPress    2ч назад  │
│ Бюджет: 25 000 – 40 000 ₽                            │
│ ▓▓▓▓▓▓▓▓▓▒  88%  Совпадение    [Брать ✓]            │
│ #wordpress  #верстка  #figma                         │
└──────────────────────────────────────────────────────┘
```

| Элемент | Стиль |
|---------|-------|
| Карточка | border `#E8E8EC`, radius `16px`, padding `20px 24px` |
| Источник | цветная точка-метка (FL=зелёная, Kwork=оранж, TG=синяя) + текст 12px/600 |
| Время | Manrope 12px muted, справа |
| Заголовок | Manrope 16px/600, `#0A0A0A`, одна строка (truncate) |
| Бюджет | Manrope 14px/400, `#3D3D3D` |
| Match-bar | высота 6px, radius 999px, фон `#E8E8EC`, fill `#2563EB`, анимируется при появлении |
| Match % | Manrope 13px/700, рядом с баром |
| AI-чип | «Брать» зелёный / «Сомнительно» серый — pill 12px/600 |
| Теги | chip-строка: `#F5F5F7` фон, `#6B6B6B` текст, 11px |
| Курсор | pointer — вся карточка кликабельна |

**Hover:** `translateY(-2px)` + тень `0 8px 24px rgba(0,0,0,0.10)` + border темнее `#C8C8D0` (150ms ease-out)

### 3.2 Раскрытое состояние (клик по карточке)

```
┌──────────────────────────────────────────────────────┐
│ [●FL.ru]  Верстка лендинга на WordPress    2ч назад  │
│ Бюджет: 25 000 – 40 000 ₽                            │
│ ▓▓▓▓▓▓▓▓▓▒  88%  Совпадение    [Брать ✓]            │
│ #wordpress  #верстка  #figma                         │
│ ─────────────────────────────────────────────────── │  ← разделитель
│ Нужна верстка лендинга под дизайн Figma.             │
│ Стек: WordPress + Elementor или чистый HTML/CSS.     │
│ Срок — 5 дней. Правки обсуждаются.                   │
│                                                      │
│ [Открыть оригинал ↗]                                 │
└──────────────────────────────────────────────────────┘
```

| Элемент | Стиль |
|---------|-------|
| Анимация раскрытия | `max-height` 0 → auto + `opacity` 0 → 1, 300ms ease-out |
| Разделитель | `1px solid #E8E8EC`, margin `12px 0` |
| Текст задания | Manrope 14px/400, `#3D3D3D`, `line-height 1.6`, max 5 строк + «…» если длиннее |
| Кнопка оригинала | ghost-кнопка: border `1px #E8E8EC`, hover border темнее, иконка `↗`, Manrope 13px/600 |

**Повторный клик** — сворачивает. Одновременно открыта только **одна карточка** (остальные закрываются).

### 3.3 В `/cabinet` — дополнительный блок

```
│ ─────────────────────────────────────────────────── │
│ 🤖 Рыночная цена: ~32 000 ₽ (медиана по похожим)   │
│                                                      │
│ [Сгенерировать отклик]    [Открыть оригинал ↗]      │
└──────────────────────────────────────────────────────┘
```

| Элемент | Стиль |
|---------|-------|
| Рыночная цена | Manrope 13px, иконка 🤖 или маленький AI-значок, `#3D3D3D` |
| «Сгенерировать отклик» | primary pill-кнопка, `#0A0A0A` фон, белый текст |
| До фазы 3f | кнопка `disabled` (opacity 0.4) + tooltip «Скоро» при hover |

## 4. Страница `/cabinet`

### 4.1 Layout

```
┌─ Header ─────────────────────────────────────────────┐
│ RawLead    Главная · Лента · ...      [Кабинет ●]    │
└──────────────────────────────────────────────────────┘
┌─ Cabinet header ─────────────────────────────────────┐
│ Мои заказы                                           │
│ [#wordpress] [#верстка] [#react] [#figma] [+ добавить тег] │
└──────────────────────────────────────────────────────┘
┌─ Sidebar (280px) ──┐ ┌─ Feed ─────────────────────────┐
│ (те же фильтры)    │ │ Карточки с блоком AI-агента    │
│                    │ │ (final_rank DESC)               │
└────────────────────┘ └────────────────────────────────┘
```

### 4.2 Cabinet header — «Мои теги»

```
Мои заказы

Ваши теги:  [#wordpress ×]  [#верстка ×]  [#react ×]  [#figma ×]  [+ Добавить тег]
```

| Элемент | Стиль |
|---------|-------|
| Заголовок | Unbounded 28px/700 |
| Тег-чип активный | `#0A0A0A` фон, белый текст, `×` удалить, pill |
| Hover на чипе | `×` ярче, opacity 0.85 |
| Добавить тег | outline pill, `+` слева, по клику — inline input появляется в строке |
| После изменения тегов | лента перезагружается (spinner 300ms) |
| Пустые теги | текст «Добавьте теги, чтобы мы подбирали заказы под вас» + кнопка «Добавить» |

### 4.3 Пустые состояния

| Ситуация | Что показать |
|----------|--------------|
| Теги есть, матчей нет | Иконка-поиск + «Подходящих заказов пока нет. Попробуйте расширить теги или проверьте позже.» |
| Нет тегов | Иконка-тег + «Добавьте теги профиля, чтобы видеть персональную ленту» + CTA |
| Загрузка | 3 skeleton-карточки |

## 6. Пульт (desktop) — визуальные доработки

**Согласовано: функционал не меняется, только визуал.**

### 6.1 Пульсация лампы в режиме running

Когда радар работает (лампа `--ok`): медленный пульс свечения.

```css
@keyframes lamp-pulse {
  0%, 100% { box-shadow: 0 0 3px 1px var(--color-status-ok-glow), 0 0 12px 5px var(--color-status-ok-glow); }
  50%       { box-shadow: 0 0 6px 3px var(--color-status-ok-glow), 0 0 22px 10px var(--color-status-ok-bloom); }
}
.lamp__dot--ok { animation: lamp-pulse 3s ease-in-out infinite; }
```

### 6.2 Вкладка «Статус» — структурированный вывод

**Это задача для Coder.** В `CODER_PROMPT` Lead Architect указывает: вкладка «Статус» рендерит данные из `/status-text` как структурированные строки (не сырой текст):

```
Радар:    ● работает   (12:34)
Биржи:    ● ok         FL.ru · Kwork
TG:       ⚠ warn       acc1 ready · acc2 join...
Последний лид:  2 мин назад
```

Формат: моноширинный, ключ выровнен по правому краю (padding-right), значение — цветной индикатор + текст. CSS-only, данные из существующего `/status-text` эндпоинта.
