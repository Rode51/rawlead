## 1. Executive summary

1. **Главная дыра** — в стопе ~30 токенов и сильный уклон в «дизайн/стек/учёба/накрутка»; **нет** массового TG-шума: инфобиз/курсы, крипто/ставки, найм в штат, «партнёры/пиши в лс +», реклама чужих каналов, опросники, серые «удалёнка без опыта», дубли без ТЗ.
2. **Убрать или заменить:** голое **`отзывы`** (риск «система отзывов на сайте») → фразы **`товар за отзыв`**, **`накрутка отзыв`**, **`кэшбэк за отзыв`**; **`модель`** → **`3d модел`** / оставить только **`3d`** в связке с визуалом.
3. **Не добавлять:** `бот`, `парсинг`, `java`, `бесплатно`, `вакансия`, `удалённо`, `опрос` (короткие) — высокий риск отсечь живые заказы.
4. **Расширить стоп до ~65–75 фраз** (секция 3): приоритет **длинные маркеры** («оформление по тк», «инфобизнес», «пиши в лс +»), не односложные.
5. **FL/Kwork vs TG:** на биржах меньше верификации чата и «папок каналов»; для TG — отдельный подсписок (секция 4), при росте шума — поле `stop_tg` в коде (задача Coder).
6. **Бюджет «за отзыв»** — стоп на уровне 2; **&lt; 3 500 ₽** — оставить ИИ + `MIN_BUDGET_RUB` (уже в PROFILE).
7. **Продажа услуг** («делаю сайты», «беру заказы») — стоп; **поиск исполнителя** («нужен сайт») — не трогать.
8. **Верификация чата** — в коде `sender.bot`; в стоп — только **имитация заказа** с «пройдите верификацию» в тексте поста (не KYC заказчика в ТЗ).
9. **Белый список «Берём»** при `FILTER_WIDE=1` почти не влияет; дополнения в секции 3 — на будущее `FILTER_WIDE=0` (логистика, API, CRM).
10. **Критерий успеха (оценка до прогона):** ручные 20 тестов (секция 6) + 50–100 реальных постов из лога TG за неделю после внедрения; цель — **&lt;5% ложных стопов** на живых, **≥70%** типового спама TG без ИИ (не измерено заранее).
11. **Следующий шаг владельцу:** вставить секцию 3 в `FILTERS.md` → перезапуск радара → неделя заметок «ложный стоп / прошёл мусор».
12. **Следующий шаг Coder (опционально):** `stop_tg` в `filters.py`, дедуп TG↔FL, эвристика «10+ хештегов» — не substring.

## 4. Стопы только для Telegram

**Внедрить в код отдельно** (`stop_tg` в `ListingWordFilter` или второй markdown-блок), если после общего списка в чатах всё ещё шум:

- `пройдите верификацию`, `правила чата:`, `нажмите кнопку под`, `заявка на вступление`, `админ разрешил публиковать`, `forwarded from уведомление`, `публикация вакансии/резюме` (служебные боты каналов)

На **FL/Kwork** эти строки почти не встречаются — не применять к биржам.

## 7. Что оставить ИИ (не дублировать стопами)

- **Сложность и стек:** «микросервисы», «с нуля маркетплейс», чистый React/Vue, mobile, **1С с нуля** — PROFILE МИМО.
- **Бюджет:** &lt; 3 500 ₽ на бирже, размытый «договоримся».
- **Качество ТЗ:** объём, сроки, адекватность; «без ТЗ» — МИМО.
- **Сомнительно:** крупная платформа, но модулями (парсер + API + WP).
- **#вакансия с разовым проектом** vs штат — контекст, не хештег.
- **SMM/маркетинг** с автоматизацией/ботом vs «ведение инстаграм руками».
- **Дубликаты** одного заказа FL↔TG (будущий дедуп).
- **Пограничный дизайн:** «лендинг + правки вёрстки» vs «логотип с нуля».
- **Юридически серая зона** без явных слов в стопе.

## Приложение: слабые места текущего FILTERS.md (до правки)

| Проблема | Риск |
|---------|------|
| Нет инфобиз/крипто/штат/каналов | Мусор идёт в ИИ → лишние TG-уведомления |
| `отзывы` в старом списке не было, но `написать отзыв` есть | Ок; не добавлять голое `отзывы` |
| `модель` в старом списке | Ложный стоп на «модель данных» — **убрать** |
| `анимация` | Может задеть «анимация интерфейса» — заменено на контекст видео |
| `бот` только в «берём» | Правильно; не переносить в стоп |
| Нет TG-верификации в тексте | Дублирует не bot-filter, а посты-модерацию |
| `MIN_BUDGET_RUB=1000` vs PROFILE 3500 | Мелочь доходит до ИИ — настройка `.env`, не стоп-слова |

*После согласования владельца: перенести секцию 3 в `FILTERS.md` или задача `@coder` по `CODER_PROMPT.md`.*
