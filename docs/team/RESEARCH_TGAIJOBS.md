# Референс: TgAiJobs (изучить позже)

Источник: публичное описание сервиса (2026-05).  
Наш продукт: [`VISION_SAAS_SITE.md`](VISION_SAAS_SITE.md), радар: фаза 0–1.

**Не копировать слепо** — у них фокус на **вакансии Digital / найм**, у нас — **фриланс-заказы** (биржи + TG, отклик исполнителя).

---

## Как у них устроено (кратко)

1. **24/7 мониторинг** списка TG-каналов/групп.
2. **AI-пайплайн на каждое сообщение:**
   - это вакансия или мусор (NLP);
   - качество (полнота, контакты);
   - **дедуп** (похожие формулировки, должность, компания);
   - **структурирование** (роль, компания, локация, формат, зарплата, контакты);
   - **skills/domains** явные и **неявные** (из описания обязанностей);
3. **Поиск/фильтры** по структурированным полям на сайте.
4. **Источники** — своя курация; пользователь может **предложить канал** админу.
5. **VIP** для постов в их официальной группе.
6. Ежедневное обновление, удаление устаревших, дообучение моделей.

## Что взять идеи для RawLead / SaaS

| Идея TgAiJobs | У нас сейчас | Когда / приоритет |
|---------------|--------------|-------------------|
| Многоступенчатый AI (мусор → качество → дедуп) | Фильтр слов + один проход ИИ | **Фаза 2** — разнести на шаги в промпте/коде |
| Структурированные поля в БД (роль, формат, зарплата…) | `leads`: title, budget, url, ai_verdict | **SaaS** — расширить схему Neon + поиск на сайте |
| Дедуп «похожих» текстов, не только id | SQLite/Neon по `(source, external_id)` | **Бэклог** — эмбеддинги или fuzzy hash |
| Неявные навыки из текста | В `PROFILE.md` / промпте частично | **ИИ v2** — поле `implicit_skills` в JSON |
| Каталог отслеживаемых каналов | `SOURCES_POOLS.md` | **Фаза 1 TG** — список + статус «слушаем» |
| «Предложить свой канал» | Нет | **SaaS** — форма → модерация |
| Удаление старых вакансий | Нет | **Позже** — TTL в Neon |
| Поиск на сайте по фильтрам | WP скелет, без БД поиска | **Этап E** — кабинет + API к Neon |

## Технические заметки (для Coder, позже)

- Real-time: у них, вероятно, **webhook/long poll** Telethon или MTProto; у нас цикл + NewMessage.
- БД: у них явно **структурированная вакансия**; нам для MVP достаточно `leads` + JSON-поле `parsed` (jsonb).
- Дедуп LLM: дорого на каждый пост — сначала **дешёвый** фильтр, потом AI на прошедших.
