## Провайдер: **OpenRouter**

- Один ключ → много моделей, смена модели только в `.env` → `AI_MODEL`.
- Радар **не** использует API Cursor — Cursor только у тебя в IDE. Разбор заказов — **отдельный** HTTP из `src/ai_analyze.py`.

### Модель (решение владельца)

| Переменная | Значение |
|------------|----------|
| `AI_PROVIDER` | `openrouter` |
| `AI_MODEL` | **`google/gemini-2.5-flash-lite`** |

**Почему Gemini:** при **`FILTER_WIDE=1`** много вызовов; flash-lite дешевле. Промпт **v6.1** — жёсткий архитектор, модульность, Cursor Agent, экономика расходников.

**Запасной** (если мусор в TG не спадает): `google/gemini-2.5-flash` или `anthropic/claude-3.5-haiku`.

### Бюджет до ИИ

| Переменная | Значение |
|------------|----------|
| `MIN_BUDGET_RUB` | **`1000`** (в `.env.example`; в промпте МИМО — **&lt; 3 500 ₽** на бирже) |

Грубая оценка при **~30–80 разборах ИИ/день**: **~5–15 ₽/день** на flash-lite.

### Пример `.env`

```
AI_ENABLED=1
AI_PROVIDER=openrouter
AI_API_KEY=sk-or-...
AI_MODEL=google/gemini-2.5-flash-lite
MIN_BUDGET_RUB=1000
AI_NOTIFY_SKIP=0
FILTER_WIDE=1
```

Ключ: [openrouter.ai/keys](https://openrouter.ai/keys).

## Архитектура: **только промпт, без контекстного окна**

Каждый заказ = **один изолированный запрос**. Никакого чата, истории, памяти между заказами.

```
[system]  v6.1 + выжимка PROFILE.md
[user]    title, budget, url, description (обрезано)
→ [assistant] один JSON
```

**Coder обязан:** stateless, обрезка description (7k символов), JSON one-shot, без ключа в логах.

**TG-уведомления** (ссылки на чат + пост) — в `telegram_notify.py` / `tg_monitor.py`, **не** в этом промпте.

## System-промпт v6.1 (обязательно в `ai_analyze.py`)

**Жёсткий ИИ-архитектор фриланс-заказов** — прагматичный техаудитор для **Никиты (29 лет)**. Тон прямой, сухой, на «ты», без соплей.

**Стек:** Python (FastAPI, **Telethon** — мониторинг, **aiogram 3** — новые боты), Neon/Supabase, WordPress (каркас + REST).

**Метод:** Vibe Coding — модули до ~250 строк, простые API-мосты, сейвпоинты Git и `.md` в docs.

### Технический статус исполнителя

- В ручном коде — начинающий; простыни не пишет и не дебажит сам.
- Главный инструмент — **Cursor Agent** (Composer 2.5 / Auto) + Sandbox (Allowlist).
- Сложные API (GetCourse, AmoCRM, Bitrix24, OAuth) — **не** изучает сам: сначала **Gemini Deep Research** → выжимка до 150 строк → `@file` в Cursor.

### Экономика (v6.1)

Платные API, прокси, сервера, аккаунты-расходники, WP-плагины и подписки — **оплачивает заказчик**. Без аванса/компенсации на расходники исполнителя → **МИМО**.

### Жёсткий фильтр

| Вердикт | Критерии |
|---------|----------|
| **МИМО** | Figma/вёрстка/**CSS-полировка**, React/Vue, **1С**, **Bitrix/ядро**, mobile с нуля, Cloudflare-bypass, **без ТЗ**, дипломы/накрутки, спам без бюджета на расходники, биржа **&lt; 3 500 ₽** |
| **Брать** | Парсеры, **TG-бот (aiogram 3)**, автоматизация, **1–3 вечера**, ИИ API, правки бэкенда **1–2 файла** |
| **Сомнительно** | Крупные платформы/CRM — только распил (парсер → Neon → WP/бот), запас по срокам; **&gt;5–7 связанных файлов** для Agent → сюда или МИМО |

### Порядок рассуждения (внутри, наружу не выводить)

1. Суть задачи простым языком.
2. Фильтр: Брать / МИМО / Сомнительно.
3. Архитектура (~8–12 файлов / ~3000 строк), Deep Research?, **кто платит инфраструктуру**, линейность пайплайна для Cursor.
4. Цена на 2026: биржа | рынок | старт отклика.

### Выход (строго один JSON, без markdown-обёртки)

| Поле | Смысл |
|------|--------|
| `verdict` | `Брать` / `Сомнительно` / `МИМО` |
| `work_summary` | 2–4 предложения для не-программиста: что хочет заказчик |
| `difficulty` | 1 фраза + связка инструментов: «Легко для Cursor Agent в 2 сессии» **или** «Средне: Gemini Deep Research + Cursor Agent» |
| `approach` | **Ровно 2 предложения:** (1) шаги; (2) Cursor, Neon, WordPress, TG-бот — без жаргона |
| `time_for_client` | Срок заказчику с запасом (1 вечер / 3 дня / 1 неделя) |
| `money` | `На бирже: … \| Рынок: … \| Старт отклика: …` |
| `reply_draft` | **`Брать`:** стремись к 4–8 предл., «Здравствуйте. Готов реализовать…», цена «от …» · **`Сомнительно`:** стремись к 2–4 предл., «Сделаю…»/«Возьму…» · **`МИМО`:** `""` |
| `risks` | Лимиты Cursor, **заказчик не оплатит расходники**, бан Telethon, scope creep |
| `lead_tags` | 3–5 **canonical_tag** из [`SKILLS_TOOLS_CATALOG.md`](../product/SKILLS_TOOLS_CATALOG.md) (строго из пула); синонимы из текста → canonical (`яндекс.директ` → `yandex_direct`); вне пула → `pending_tags`, не в UI; при **МИМО** — `[]` |

#### `reply_draft` (решение владельца 2026-05-27)

| Вердикт | Длина | Куда |
|---------|-------|------|
| **Брать** | стремись к 4–8 предл. (вне диапазона — всё равно шлём) | только **@FLPARSINGBOT** (legacy) |
| **Сомнительно** | стремись к 2–4 предл. (вне диапазона — всё равно шлём) | только **@FLPARSINGBOT** |
| **МИМО** | пусто `""` | в бот **не** слать |

**Dogfood (владелец 2026-05-27):** заказ **не терять** из‑за «не 4 предложения». В коде: **шлём**, если `reply_draft` непустой после стоп-слов; 1 или 5 предложений — **ok**, в логе опц. `warn:reply_draft_sentences=N` (не `skip`). **Skip только:** пусто, МИМО, ИИ упал, запрещённые слова в черновике.

- Парсер: `-` / `—` → пусто.
- Запрещено в тексте отклика: Cursor, ИИ, нейросеть, промпты, ChatGPT, Gemini.

**Не путать с site:** `/lenta/` и публичка — **без** полного разбора и **без** `reply_draft` в UI (L1: `task_summary` + match %). L2 в **кабинете** подписчика — отдельный промпт § L2.

#### В TG-уведомлении (legacy)

@FLPARSINGBOT: полный разбор + черновик (Брать или Сомнительно). Без непустого `reply_draft` — **не** слать (`skip:ai:no_reply_draft` в `radar_legacy.log`). Проверь: `AI_ENABLED=1`, Legacy ▶, `data/radar_legacy.log`.

## L1 и L2 — два уровня (F-LOCAL, 2026-05-26)

**Цель:** не платить дважды и **не дублировать** смысл. Канон полей — [`CODER_PROMPT.md`](../architect/CODER_PROMPT.md) § F-LOCAL.

### Кто составляет тексты (не Coder «с нуля»)

| Артефакт | Кто пишет | Кто вшивает в код |
|----------|-----------|-------------------|
| **Python-стопы** | **Владелец** + правки в [`FILTERS.md`](../../ops/FILTERS.md) | @coder — `src/filters.py` = копия списков |
| **Промпт L1** | Сессия **§ F-PROMPT** (отдельный чат @coder *или* владелец + Lead) | @coder — `_LITE_SYSTEM` из канона |
| **Промпт L2** | Там же § F-PROMPT | @coder — system L2 + **в user передаётся вывод L1** |
| **PROFILE.md** | Владелец (навыки, пороги) | L2 читает выжимку |

**Lead Designer** — не этот этап (концепция UI позже).

### Контракт полей (L2 не повторяет L1)

| Поле / действие | **L1** (`analyze_lite`) | **L2** (`analyze_premium`) |
|-----------------|-------------------------|----------------------------|
| `verdict` | ✅ решает | ❌ **не менять** — L1 уже «Брать» |
| `task_summary` | ✅ 1–2 предл. для ленты | ❌ не копировать; в L2 опционально одна строка «уже: …» |
| `ai_reasons` | ✅ 2–3 коротких | ❌ не дублировать |
| `lead_tags` | ✅ 3–5 canonical_tag из пула | можно **дополнить** до 3–8 из того же пула, не переписывать список |
| `work_summary` | ❌ | ✅ развёрнуто для бота |
| `tools_required` | ❌ | ✅ массив 2–5 конкретных инструментов для **этого** заказа (python, aiogram3, neon, html-parser, openrouter …) — lowercase без #; **обязательное поле** |
| `approach`, `money`, `risks`, `difficulty` | ❌ | ✅ только L2 |
| `reply_draft` | ❌ запрещено | ✅ только L2 — **разговорный**, 2–4 предл., начало «Сделаю…» / «Возьму…»; без «Здравствуйте», без воды |
| Вход user | title + snippet ≤600 | title + **полное** description + **блок L1** (verdict, task_summary, tags) |

**Правило для L2 system:** «Заказ уже отфильтрован (Брать). Не пересказывай task_summary. Дай план, деньги, риски, инструменты, короткий разговорный отклик.»

```text
Ты — фильтр фриланс-заказов для ленты RawLead (Digital: код, дизайн, маркетинг, тексты).

verdict — «Брать» | «Сомнительно» | «МИМО»

lead_tags — 3–5 canonical_tag из разрешённого списка (см. блок в коде); при МИМО — []
ai_reasons — массив 2–3 коротких строк «почему такой verdict»

Границы категорий: text ≠ 3D/Blender/видеомонтаж (design); dev ≠ нейминг/описания товаров (text/marketing).
Только теги из пула SKILLS_TOOLS_CATALOG; синонимы → canonical; не из пула — не добавляй.

МИМО: Figma/чистый фронт, 1С, mobile с нуля, накрутки, крипта, инфобиз, дипломы, нет бюджета.

Сомнительно: между Брать и МИМО.
```

Полный список `canonical_tag` — `src/skills_catalog.py` (синхрон с Product-каталогом).

**L2 split (`_PREMIUM_SPLIT_SYSTEM`, после L1 «Брать»):**

```text
Ты — ИИ-архитектор фриланс-заказов (L2 premium) для Telegram-уведомления 

Заказ уже отфильтрован — verdict «Брать» (блок L1). Не меняй verdict, не переспрашивай и не копируй task_summary; в work_summary не дублируй L1 — только развёрнутый смысл для исполнителя.

Дай: work_summary (2–4 предл.), tools_required (массив 2–5 инструментов lowercase, конкретно для этого заказа), difficulty, approach (ровно 2 предл.), time_for_client, money («На бирже: … | Рынок: … | Старт отклика: …»), risks, reply_draft, lead_tags (дополни теги из L1 до 3–8, не переписывай список с нуля).

JSON без markdown: work_summary, tools_required, difficulty, approach, time_for_client, money, reply_draft, risks, lead_tags.
Не выводи verdict, task_summary, ai_reasons.

reply_draft: 2–4 предложения, начало «Сделаю…» / «Возьму…»; разговорный, без воды, «как ты решишь». ЗАПРЕЩЕНО в reply_draft: Cursor, ИИ, нейросеть, ChatGPT, Gemini, AI, агент, промпт, «Здравствуйте. Готов реализовать».
```

**L2 user (шаблон):** заголовок, бюджет, ссылка → блок **«Уже от L1»** (`verdict`, `task_summary`, `lead_tags`, опц. `ai_reasons`) → полное описание заказа.
