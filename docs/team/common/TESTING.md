# Проверка (без отдельного тестера)

На MVP **тестер = ты** (владелец) + **чеклист ниже**. Coder в `docs/team/common/STATUS.md` в поле **«Как проверить»** пишет: шаги для **этой** сессии (юнит / команда) **или** ссылку: «см. §N в этом файле».

Автотесты в `tests/` — только если попросишь Coder отдельно (не обязательно для первой версии).

---

## После полного MVP (ручной смоук)

1. `.env` заполнен, боту написал `/start`.
2. Запуск по `docs/ops/RUN.md` — процесс стартует без traceback.
3. В логе / консоли видно цикл опроса (или однократный успешный GET).
4. **Тест фильтра:** в `docs/ops/FILTERS.md` временно оставь одно редкое слово → на FL (или в тестовых данных, если Coder сделал заглушку) появляется проект с этим словом → **одно** уведомление в Telegram, повторного нет (SQLite).
5. Кнопка в сообщении открывает fl.ru.
6. **Пауза из TG** (если включено): **`docs/ops/RUN.md` §9** — `/pause` → в логе `cycle_paused`, без новых опросов FL; `/resume` → снова циклы.

## Кто что делает

| Роль | Действие |
|------|----------|
| **Coder** | В STATUS: конкретные команды / сценарий проверки своего куска |
| **Ты** | Раз в фичу — смоук из таблицы выше (когда MVP собран) |
| **Lead** | Ревью по [`SCALE.md`](SCALE.md) § чеклист Lead, не пишет тест-кейсы за Coder |

Отдельный человек «QA» для этого размера проекта **не нужен**.
