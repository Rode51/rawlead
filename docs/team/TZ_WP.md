# ТЗ — WordPress «пульт» (фаза 3, стартуем планирование)

Версия: **0.1** · Lead · 2026-05-20

**Решение:** WP — **экран настроек и аналитики**, движок — **Python** (радар на ПК → позже VPS), данные — **PostgreSQL в облаке** или **SQLite на ПК** (см. § «База»).

**База (если Supabase Free кончился):**

| Вариант | Цена | Для пульта |
|---------|------|------------|
| **SQLite на ПК** (`data/projects.db` + таблица лидов) | 0 ₽ | MVP без облака; WP → через маленький API на ПК или отложить WP |
| **[Neon](https://neon.tech)** Postgres | Free tier | **Замена Supabase для Python**; WP → HTTP API к радару, не напрямую в Neon (у shared-хостинга обычно нет Postgres) |
| **[Turso](https://turso.tech)** | Free | SQLite в облаке, другой стек |
| **Supabase** | Пауза проекта / апгрейд | Старый план, если снова откроют Free |

Рекомендация Lead при нулевом бюджете: **сначала SQLite + радар на ПК**; WP-пульт подключать, когда будет Neon **или** VPS.

WP **не** парсит FL/TG сам (не PHP-парсеры).

---

## 1. Архитектура (дешёвый вариант)

```
[Ты в браузере] → WordPress на дешёвом хостинге
                        ↓ читает/пишет
                   Supabase (облако, $0 старт)
                        ↑ читает настройки, пишет лиды
[Радар Python] на твоём ПК (позже VPS 24/7)
        FL / Kwork / TG (Telethon)
                        ↓
              Telegram-бот → ты
```

Так **не нужен дорогой VPS** на первом этапе: WP живёт на shared-хостинге, радар — на ПК как сейчас.

---

## 2. MVP пульта (что в первой версии)

| Экран WP | Действие |
|----------|----------|
| Вход | Только ты (логин/пароль WP) |
| Источники | Вкл/выкл FL, Kwork; позже — чаты TG |
| Фильтры | Ссылка на правку или простые поля (стоп-слова) |
| Куда слать | Chat ID / «в бота» (пока одно значение) |
| Лиды | Таблица последних заказов из Supabase |
| Статус | «Радар на ПК: онлайн/офлайн» (позже heartbeat) |

**Не в MVP:** оплата, мультиюзеры, красивая аналитика графиками.

## 3. Что нужно от владельца (по шагам)

### Шаг 1 — Supabase (бесплатно, 15 мин)

1. [supabase.com](https://supabase.com) → проект `fl-radar` (регион ближе к РФ по возможности).
2. Сохранить в **личный файл** (не в Git): URL, `anon` key, `service_role` key.
3. Сообщить Lead: «Supabase создан» (ключи в чат **не** слать — только «готов»).

### Шаг 2 — Хостинг под WordPress (платно, но дёшево)

Нужно: **PHP 8+**, **MySQL/MariaDB**, SSL, установка WordPress в 1 клик.

См. раздел **«Хостинг»** ниже — ориентир **150–350 ₽/мес**.

1. Купить тариф **«Сайт» / WordPress»** (не VPS, если экономим).
2. Привязать домен: свой (~200–500 ₽/год) или поддомен хостера (`xxx.beget.tech`) — **бесплатно**.
3. Установить WordPress, задать **сильный пароль** админа.
4. Сообщить Lead: адрес сайта (URL), «WP стоит».

### Шаг 3 — Решения от тебя (ответить Lead)

| Вопрос | Варианты |
|--------|----------|
| Домен | Свой / поддомен хостера |
| Кто заходит в пульт | Только ты |
| Язык админки WP | Русский |
| Что важнее в MVP | (а) список лидов (б) вкл/выкл источники (в) оба |

### Шаг 4 — Параллельно (не блокирует WP)

- API_ID с my.telegram.org (когда заработает).
- Список чатов → `docs/ops/SOURCES_POOLS.md`.
- Радар на ПК: `python src/main.py`.

### Шаг 5 — После Coder

- Проверить: в WP виден тестовый лид; переключатель «FL вкл» сохраняется в Supabase; радар на ПК подхватывает настройку.

## 4. Хостинг — как дешевле

| Вариант | Цена ориентир | Плюсы | Минусы |
|---------|---------------|-------|--------|
| **Shared + WP** (Beget, Timeweb, Reg.ru, Sprinthost…) | **~150–350 ₽/мес** | Дёшево, WP в 1 клик | Python 24/7 там обычно **нельзя** |
| **Один дешёвый VPS** | **~300–600 ₽/мес** | WP + Python в одном месте позже | Настройка сложнее, не «в 1 клик» |
| **Только ПК, без WP** | **0 ₽** | Уже работает | Нет пульта в браузере |

**Рекомендация Lead (минимум денег):**

1. **Shared-хостинг** с акцией «первый месяц 1 ₽» (часто Beget/Timeweb) + **бесплатный поддомен** хостера.
2. **Supabase Free** — лиды и настройки.
3. Радар **пока на ПК**; VPS — когда понадобится 24/7 без включённого компьютера.

Сравни 2–3 сайта по словам «хостинг WordPress недорого» / «тариф Start» — смотри: **SSL бесплатно**, **объём 1–5 ГБ**, **MySQL**.

**Не брать** на старте: конструкторы (Tilda/Wix) вместо WP — пульт не соберём; дорогой VPS «на вырост».

---

## 5. Очередь Coder (после согласования Lead)

1. Supabase: таблицы `settings`, `leads`, `sources`.
2. Python: запись лидов с бирж в Supabase (расширение `main.py`).
3. WP: простой плагин или child-theme — читает Supabase REST, форма настроек.
4. Док: `docs/ops/WP_RUN.md` — как зайти в пульт.

Зависимости: **не ждёт** TG, но **ждёт** Supabase от владельца.

## 6. Приёмка MVP пульта

- [ ] Зайти на URL WP под своим логином.
- [ ] Видны последние лиды с FL/Kwork (после цикла радара).
- [ ] Переключатель источника сохраняется и влияет на радар после перезапуска/цикла.
- [ ] Секреты Supabase не в Git.
