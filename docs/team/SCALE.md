# Масштаб — как вести большой проект (соло + AI)

**Один цикл = одна маленькая задача.** Большой продукт = **20–50 сданных циклов**, не один гигантский чат.

Связано: [`HOW_TO_USE_CURSOR.md`](HOW_TO_USE_CURSOR.md) · [`LEAD.md`](LEAD.md) · [`../ops/BACKUP.md`](../ops/BACKUP.md)

---

## Цикл (обязательный)

| № | Кто | Действие | Артефакт |
|---|-----|----------|----------|
| 1 | **Ты / Gemini** | Идея, черновик (дешево, в браузере) | 5–10 строк → Lead |
| 2 | **Lead** | Кусок в фазу ROADMAP; **одна** задача 1–2 дня | `TASKS.md` + **`CODER_PROMPT.md`** |
| 3 | **Coder** | **Новый чат** → код | `STATUS.md` (Сделано / Файлы / Как проверить) |
| 4 | **Ты** | Запуск по `RUN.md`, **~5 мин** приёмка | ок / не ок |
| 5 | **Lead** | Ревью → закрыть TASKS, удалить промпт, FOR_YOU, архив | `TASKS_HISTORY.md` |
| 5b | **Ты** | **Бэкап** после принятой задачи | [`BACKUP.md`](../ops/BACKUP.md) |
| 6 | Повтор | Следующая задача из ROADMAP | — |

**Не ок на шаге 4:** Lead → `docs/problems/` → Mechanic → снова Lead (не чинить в Coder-чате старой задачи).

---

## Жёсткие правила

### Coder — новый чат на каждую задачу

| Правило | Почему |
|---------|--------|
| Одна задача = **один** новый чат Coder | Чистый контекст, меньше галлюцинаций |
| После сдачи чат **не переиспользовать** | Старый контекст путает следующую фичу |
| Без `CODER_PROMPT.md` — **не начинать** | Lead не поставил задачу |
| Coder **не правит** TASKS / FOR_YOU | Только Lead |

### Lead — размер задачи

| OK | Не OK |
|----|-------|
| «acc2 подключается, лог старт» | «Сделай весь multi-session» |
| «Кнопка паузы в боте» | «Переделай архитектуру TG» |
| 1–2 дня Coder | «На неделю всё сразу» |

**Одна активная задача Coder** в `TASKS.md`. Бэклог — список без промпта.

### Одна активная фаза

В `TASKS.md` в работе — **одна линия** (сейчас фаза 1 TG). WP / SaaS — только бэклог, пока Lead не откроет фазу.

### Документы — иерархия

| Уровень | Файл | Когда менять |
|---------|------|--------------|
| Зачем | `PRODUCT_VISION.md` | редко |
| Когда | `team/ROADMAP.md` | раз в «спринт» / фазу |
| Сейчас | `FOR_YOU.md`, `STATUS.md` | каждый цикл |
| Задача | `CODER_PROMPT.md` | один цикл, потом удалить |

## Чеклист Lead после сдачи Coder

- [ ] `STATUS.md` — факты совпадают с кодом (не верить на слово)
- [ ] `TASKS.md` — задача снята или перенесена
- [ ] `CODER_PROMPT.md` — **удалён**
- [ ] `archive/TASKS_HISTORY.md` — одна строка
- [ ] `FOR_YOU.md` — один следующий шаг владельцу
- [ ] CSV / ops — без противоречий с STATUS
- [ ] Напомнить владельцу: **бэкап** ([`BACKUP.md`](../ops/BACKUP.md))
- [ ] Секреты только в `.env`, не в repo
- [ ] Интервал опроса FL ≥ 10 мин; дедуп SQLite
- [ ] Инцидент (если был): тикет `docs/problems/*.md` — **решено** + раздел «Решение»
- [ ] `docs/ops/RUN.md` актуален для владельца

---

## Чеклист владельца (шаг 4–5b)

- [ ] Команды из `STATUS.md` / `RUN.md` — **`.venv\Scripts\python.exe`**
- [ ] 5 мин: работает / не работает (не «кажется ок»)
- [ ] Принял → Lead закрывает цикл
- [ ] **Бэкап** — раз в неделю минимум; после каждой принятой фичи — желательно
